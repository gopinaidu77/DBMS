Database Management Systems (BS406) - Important Questions and Answers

Unit – I: Introduction to DBMS and Relational Algebra

Short Answer Questions

1. What is a Database?

A database is a collection of organized data that can be easily accessed, managed, and updated.

It allows storing and retrieving information efficiently. It supports multiple users and applications.

2. List any two advantages and two disadvantages of DBMS.

Advantages: Data Redundancy control, Data Integrity.

Disadvantages: High cost, Complex system to manage.

3. What is the ANSI-SPARC architecture?

It is a three-level database architecture: External, Conceptual, and Internal. It separates user views from physical storage.

4. Define Data Models with examples.

Data Models define how data is connected and accessed. Examples: Hierarchical, Network, Relational models.

5. What are the components of a DBMS?

Components include DBMS Engine, Query Processor, Metadata Catalog, and Application Interface.

6. Define the Relational Model.

A relational model organizes data into tables (relations) using rows and columns.

It uses keys and constraints.

7. What are Integrity Constraints?

These are rules to maintain data accuracy and consistency, like Primary Key, Foreign Key, and Check constraints.

8. Write short notes on Aggregation and Grouping.

Aggregation combines data using functions like SUM, AVG.

Grouping groups data using GROUP BY clause.

Long Answer Questions

1. Explain file-based systems and compare them with DBMS.

File-based systems store data in separate files. They lack security, data integrity, and concurrent access support.

DBMS solves these issues by using a centralized system with constraints, multi-user access, transaction support, and better storage mechanisms.

It enhances consistency and avoids redundancy.

2. Describe the three-level architecture of DBMS.

The three levels are:

• External level: user view.

• Conceptual level: logical structure.

• Internal level: physical storage.

This separation allows independence between data structure and storage.

3. Discuss different data models in DBMS.

Data models describe how data is organized. Hierarchical model uses parent-child relationships. Network model allows complex relationships.

Relational model uses tables. Object-oriented model integrates object features.

4. Explain various database languages supported by DBMS.

• DDL (Data Definition Language): Defines structure.

• DML (Data Manipulation Language): Handles data.

• DCL (Data Control Language): Manages permissions.

• TCL (Transaction Control Language): Manages transactions.

Each type plays a vital role in handling and securing data efficiently.

5. What are relational algebra operations? Explain with examples.

They include Select, Project, Union, Set Difference, Cartesian Product, Rename, Join, and Division.

These operations help retrieve and manipulate data from relational databases.

For example, Select filters data while Join combines data from multiple tables.

6. Explain Join and Division operations in relational algebra.

Join combines two relations on a common attribute to retrieve meaningful information.

Division is used when we need results that match all values in a related set, such as finding students enrolled in all courses.

7. Explain integrity constraints in detail.

They include:

• Primary Key: Uniquely identifies a record.

• Foreign Key: Links two tables.

• Check: Enforces a condition.

• Not Null: Prevents null values.

These constraints ensure consistency, accuracy, and proper linking of records.

8. What are the advantages and limitations of DBMS approach?

Advantages:

• Data integrity and security

• Easy data access and backup

• Reduces redundancy

Limitations:

• High cost of software and hardware

• Complexity in handling

Unit – II: SQL and Advanced SQL

Short Answer Questions

1. Write a simple SQL query to retrieve names from a student table.

SELECT name FROM student;

This query selects all names from the "student" table.

2. What is the use of GROUP BY clause in SQL?

GROUP BY is used to group rows with the same values in specified columns and allows aggregate functions like COUNT, SUM.

3. Define Subqueries with an example.

A subquery is a query within another SQL query.

Example: SELECT * FROM emp WHERE dept_id = (SELECT dept_id FROM dept WHERE name='Sales');

4. What is the difference between EXISTS and NOT EXISTS?

EXISTS checks if subquery returns any rows.

NOT EXISTS checks if it returns none.

5. What is a View? How is it created in SQL?

A view is a virtual table. Syntax: CREATE VIEW view_name AS SELECT ...

6. Define Domain Constraints and Entity Integrity.

Domain constraint restricts values to specific data types. Entity Integrity ensures primary key is not NULL.

7. What is the use of the CHECK constraint?

It limits the value range that can be placed in a column, e.g., CHECK (age >= 18).

8. What are Stored Procedures?

Stored Procedures are precompiled SQL programs stored in the DB that can be executed repeatedly.

Long Answer Questions

1. Explain DDL, DML, and DCL statements in SQL with examples.

• DDL (CREATE, ALTER, DROP): Defines schema.

• DML (INSERT, UPDATE, DELETE): Modifies data.

• DCL (GRANT, REVOKE): Controls access.

Each plays a vital role in managing and securing data in DBMS with clear syntax and defined outcomes.

2. Describe the use of aggregate functions and GROUP BY in SQL.

Aggregate functions: SUM, AVG, COUNT, MAX, MIN.

GROUP BY groups data rows based on one or more columns. Example: SELECT dept, COUNT(*) FROM emp GROUP BY dept;

helps in departmental employee counts and summary reporting.

3. Discuss how to create, modify, and delete views in SQL.

• Create: CREATE VIEW emp_view AS SELECT name FROM emp;

• Update: CREATE OR REPLACE VIEW ...

• Delete: DROP VIEW emp_view;

Views are useful for security and simplifying complex queries by abstracting the base table details.

4. Explain integrity constraints and their types with examples.

• Primary Key: emp_id INT PRIMARY KEY

• Foreign Key: FOREIGN KEY (dept_id) REFERENCES dept(dept_id)

• Check: CHECK (salary > 0)

• Not Null: name VARCHAR(20) NOT NULL

These ensure the reliability and accuracy of data.

5. Write a detailed note on advantages and disadvantages of Views.

Advantages:

• Simplify complex queries

• Enhance security

• Provide logical data independence

Disadvantages:

• May be non-updatable

• Can impact performance

They are best suited for data abstraction and control.

6. Describe the process of creating and dropping indexes in SQL.

• Create: CREATE INDEX idx_name ON table(col);

• Drop: DROP INDEX idx_name;

Indexes speed up data retrieval but may slow insert/update operations. Proper index design is crucial.

7. Explain Cursors, Triggers, and Exceptions with examples.

• Cursor: Allows row-by-row processing. DECLARE CURSOR ...

• Trigger: Runs automatically on insert/update/delete. CREATE TRIGGER ...

• Exception: Handles errors using TRY-CATCH blocks or DB exception handling.

These enhance data control.

8. Write and explain an SQL program using Functions and Packages.

A Function is used for computation. Example:

CREATE FUNCTION calc_bonus(salary NUMBER) RETURN NUMBER IS

BEGIN

RETURN salary * 0.10;

END;

Packages group procedures, functions, and variables. Example: CREATE PACKAGE employee_ops AS ...

Unit – III: ER Modeling and Normalization

Short Answer Questions

1. Define Entity and Entity Set.

An entity is a real-world object. An entity set is a group of similar entities.

Example: A student is an entity, and all students form an entity set.

2. What is a Relationship Type?

It represents an association between entities. Example: A student 'enrolls' in a course – ‘enrolls’ is the relationship type.

3. Differentiate between Strong and Weak Entity.

A strong entity has a primary key.

A weak entity cannot exist without a strong entity and uses a foreign key.

4. Define Key and Attribute in ER Model.

Key uniquely identifies an entity.

Attributes describe properties of an entity like name, age, etc.

5. What are Structural Constraints?

They define cardinality (1:1, 1:N, N:M) and participation (total or partial) in relationships.

6. Define Functional Dependency.

A functional dependency X → Y means if two tuples agree on X, they must agree on Y. It’s used in normalization.

7. What is Normalization?

Normalization is a process to reduce data redundancy and improve data integrity by organizing data into tables.

8. What is 3NF?

A relation is in 3NF if it is in 2NF and no transitive dependency exists between non-prime attributes.

Long Answer Questions

1. Explain different types of attributes in ER Model.

Attributes can be:

• Simple (indivisible)

• Composite (divided into sub-parts)

• Derived (calculated)

• Multi-valued (multiple values)

They help in fully defining an entity’s properties and relationships.

2. What are ER diagrams? Describe symbols used.

ER diagrams represent entities, relationships, and attributes.

Rectangles = entities, Diamonds = relationships, Ovals = attributes. Lines connect elements. Double lines = total participation.

This diagram helps in visualizing DB design.

3. Discuss Functional Dependencies with examples.

A → B implies B is functionally dependent on A. Example: Student_ID → Student_Name.

Functional dependencies help detect redundancy and guide normalization.

4. Differentiate between Partial and Transitive Dependency.

Partial Dependency: Non-key depends on part of composite key (1NF to 2NF).

Transitive Dependency: A → B and B → C implies A → C (2NF to 3NF).

These need to be removed for normalized DB design.

5. Explain the purpose and process of normalization.

Normalization reduces redundancy and anomalies. It involves organizing data into normal forms: 1NF (atomic), 2NF (no partial), 3NF (no transitive), BCNF (more strict).

Each form improves structure and integrity.

6. What is the difference between 1NF, 2NF, and 3NF?

• 1NF: Atomic values

• 2NF: No partial dependencies

• 3NF: No transitive dependencies

Each step removes redundancy and improves clarity of data storage.

7. Explain problems like Fan Traps and Chasm Traps in ER modeling.

Fan Trap occurs when a single entity is related to multiple entities without proper linkage.

Chasm Trap happens when a relationship is missing, preventing data retrieval. These are resolved using correct relationship mapping.

8. Describe the Database Design Methodology using ER to Relational.

Steps:

• Identify entities and relationships

• Define primary and foreign keys

• Convert attributes to columns

• Normalize tables

This method ensures a logical, efficient relational schema.

Unit – IV: Transaction Management and Concurrency Control

Short Answer Questions

1. What is a Transaction?

A transaction is a sequence of operations performed as a single logical unit of work.

It must follow ACID properties to ensure database consistency.

2. Define ACID Properties.

ACID stands for Atomicity, Consistency, Isolation, and Durability. These ensure safe and reliable transactions in DBMS.

3. What is Atomicity?

Atomicity ensures that all operations in a transaction are completed;

if not, the transaction is aborted and changes are rolled back.

4. What is a Schedule?

A schedule is an ordered list of operations from a set of transactions.

It defines the execution order in a multi-user environment.

5. What is Serializability?

Serializability ensures that a schedule results in the same database state as some serial execution of transactions.

6. Define Deadlock.

Deadlock occurs when two or more transactions wait indefinitely for each other to release resources.

7. What is Two-Phase Locking Protocol?

It is a concurrency control method that divides a transaction into two phases: growing (acquire locks) and shrinking (release locks).

8. What is Concurrency Control?

Concurrency control manages simultaneous execution of transactions to prevent conflicts and ensure consistency.

Long Answer Questions

1. Explain ACID properties with examples.

• Atomicity: All steps must succeed or fail as a whole. Example: Money transfer.

• Consistency: Ensures database remains valid after transaction.

• Isolation: Concurrent transactions do not interfere.

• Durability: Once committed, changes are permanent. These properties protect database integrity during crashes and errors.

2. Describe different states of a transaction.

Transaction states include:

• Active: Performing operations

• Partially Committed: Last statement executed

• Committed: Changes saved

• Failed: Error occurred

• Aborted: Rolled back These states help in managing transaction lifecycle effectively.

3. What is Serializability? Explain types.

Serializability ensures consistency in concurrent transaction execution. Types:

• Conflict Serializability: Based on non-conflicting operations.

• View Serializability: Based on data views. Both types aim to ensure correctness without necessarily using serial order.

4. Discuss Concurrency Control Techniques.

• Lock-based Protocols: Use shared/exclusive locks.

• Timestamp-based Protocols: Use timestamps for ordering.

• Optimistic Concurrency Control: Assumes low conflict, checks at commit. Each technique balances performance and consistency.

5. Explain Two-Phase Locking Protocol with example.

It has two phases:

• Growing phase: Transaction acquires all needed locks.

• Shrinking phase: Locks are released and no new locks can be acquired.

Example: T1 locks A and B, performs operations, then releases. Ensures serializability but may lead to deadlocks.

6. Describe Deadlock and its prevention methods.

Deadlock occurs when transactions hold and wait on resources.

Prevention methods:

• Timeouts: Abort after wait time

• Wait-Die / Wound-Wait: Based on priority

• Deadlock Detection and Recovery: Periodic checks and rollback Proper handling ensures system availability.

7. Compare Pessimistic and Optimistic Concurrency Control.

• Pessimistic: Uses locking, assumes conflict is likely.

• Optimistic: Assumes low conflict, checks at end. Pessimistic is safer but slower. Optimistic is faster but needs rollback mechanism.

Choice depends on workload nature.

8. What is Transaction Logging and Checkpointing?

• Logging: Records all transaction activities to a log file.

• Checkpointing: Saves current state to minimize recovery time.

These help in recovering the database to a consistent state after a crash.
